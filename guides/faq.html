Domande frequenti (FAQ)

Per le FAQ specifiche sulle modalità di esecuzione, consulta la guida FAQ sulle modalità di esecuzione di Qiskit Runtime.

Come cito Qiskit nella mia ricerca?​

Cita Qiskit utilizzando il file BibTeX incluso.

Come cito un computer IBM Quantum nella mia ricerca?​

Per i paper di ricerca, incoraggiamo gli autori a ringraziare IBM Quantum® usando:

Riconosciamo l'utilizzo dei servizi IBM Quantum per questo lavoro. Le opinioni espresse sono quelle degli autori e non riflettono la politica ufficiale o la posizione di IBM o del team IBM Quantum.

I riferimenti bibliografici del paper devono usare la seguente citazione:

IBM Quantum. https://quantum.cloud.ibm.com/, AAAA

Le unità di elaborazione quantistica nel paper devono essere referenziate con il loro nome univoco (ad esempio, ibmq_vigo) e facoltativamente aggiungendo la versione (ad esempio, ibmq_vigo v1.0.2). Incoraggiamo anche a fare riferimento al processore. Ad esempio:

In questo paper abbiamo usato ibmq_vigo, che è uno dei processori IBM Quantum Canary.

Esempio di citazione di un programma IBM Quantum:

IBM Quantum (2025). Estimator primitive (Versione x.y.z) [software per computer]. https://quantum.cloud.ibm.com/

Come cito IBM Quantum Composer nella mia ricerca?​

Esempio di citazione per IBM Quantum Composer:

IBM Quantum Composer. 2025. url: https://quantum.cloud.ibm.com/composer

Non vedo qualcosa nell'interfaccia utente, come le mie attività di amministrazione.​

Prova a scegliere un account diverso dal selettore di account nell'intestazione dell'interfaccia di IBM Quantum Platform.

Uso solo il piano Open, ma ho dovuto inserire la mia carta di credito. Mi verranno addebitati dei costi?​

Non ti verranno addebitati costi a meno che tu non crei un'istanza Pay-As-You-Go e la utilizzi per inviare carichi di lavoro alle QPU. Consulta Gestisci i costi per le istruzioni su come impostare un limite di spesa e per scoprire altri modi per gestire i costi.

Qual è la differenza tra una chiave API e un token API?​

Non c'è differenza. I termini chiave API e token API vengono usati in modo intercambiabile. Vedrai entrambi i termini nella documentazione perché interfacce diverse usano termini diversi.

Qual è la differenza tra un token API e un bearer token?​

Un token API è una credenziale a lunga durata usata per autenticarsi al servizio Qiskit Runtime. Un bearer token è una credenziale temporanea usata per autenticarsi all'API REST in un ambiente non attendibile. Un bearer token non può essere usato per autenticarsi al servizio Qiskit Runtime.

Devo scegliere la regione in cui vivo?​

No. Puoi usare entrambe le regioni. La regione determina dove vengono eseguiti i job e dove vengono conservati i dati. Inoltre, puoi vedere e usare solo le istanze create nell'istanza a cui hai effettuato l'accesso. Nota che solo la regione us-east supporta il piano Open.

Posso connettermi tramite un server proxy?​

Sì, ma devi usare Qiskit Runtime v0.44.0 o versione successiva.

Qual è la politica di conservazione dei dati?​

IBM conserva i tuoi dati di job e utilizzo (non eliminati) per tre anni mentre continui a usare il servizio Qiskit Runtime.

Cosa è successo alle distribuzioni Python come Qiskit Aer, Qiskit Nature e Qiskit Experiments?​

Dalla release di Qiskit SDK v1.0, il comando pip install qiskit installerà solo il Qiskit SDK. Ciò significa che alcuni moduli che hai usato in passato, come qiskit-aer o qiskit-nature, devono ora essere installati separatamente.

Prima della release di Qiskit SDK v1.0, Qiskit era composto da molte distribuzioni Python diverse, come qiskit-terra, il nucleo del compilatore; qiskit-aer, il simulatore ad alte prestazioni; il provider originale IBM Quantum®; e diversi pacchetti ora obsoleti che fornivano particolari funzionalità algoritmiche esplorative o di esecuzione di esperimenti.

Questi pacchetti software separati furono combinati in quello che veniva chiamato il metapacchetto Qiskit, che permetteva agli utenti di installare tutti questi componenti insieme tramite pip install qiskit. Questa architettura a metapacchetto è stata rimossa con la release di Qiskit SDK v1.0, e ora ciascuna delle singole distribuzioni software deve essere installata separatamente.

Se vuoi installare i pacchetti precedentemente inclusi nel metapacchetto Qiskit, visita l'ecosistema Qiskit per trovare una gamma di pacchetti adatti alle tue esigenze. Puoi anche leggere la guida alla migrazione v1.0 per ulteriori informazioni su questo cambiamento.
