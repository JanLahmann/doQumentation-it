Configuri il server Jupyter utilizzato per eseguire il codice Python nei tutorial.

Ambiente attuale:

Account IBM Quantum

Nota di sicurezza: Le credenziali sono memorizzate nel localStorage del suo browser in testo semplice. Non sono crittografate e possono essere lette dalle estensioni del browser o da chiunque abbia accesso a questo dispositivo. Utilizzi l'impostazione di scadenza qui sotto per limitare l'esposizione, e elimini le credenziali quando ha finito. Per computer condivisi o pubblici, preferisca il metodo manuale save_account() descritto di seguito.

Inserisca le sue credenziali IBM Quantum una volta qui. Saranno iniettate automaticamente tramite save_account() all'avvio del kernel, quindi non è necessario inserirle in ogni notebook. Questo si applica solo all'esecuzione di codice incorporata su questo sito — l'apertura di un notebook in JupyterLab richiede la chiamata manuale di save_account().

  1. Registrati su quantum.cloud.ibm.com/registration — nessuna carta di credito richiesta per i primi 30 giorni
  2. Accedi su quantum.cloud.ibm.com
  3. Istanza — Crei un'istanza Open Plan gratuita su Istanze se non ne ha ancora una
  4. Token API — Clicchi sulla sua icona profilo (in alto a destra), poi "API token". Copi la chiave.
  5. CRN — Copi la stringa CRN dalla sua pagina Istanze

Per passaggi dettagliati, consulti la guida Configura autenticazione di IBM (passaggio 2).

Alternativa: Eseguire save_account() manualmente in una cella del notebook

Se preferisce non memorizzare le credenziali in questo browser, incolli questo in una cella di codice ed eseguala. Le credenziali vengono salvate nell'archivio temporaneo del kernel Binder e perse alla fine della sessione.

from qiskit_ibm_runtime import QiskitRuntimeService
QiskitRuntimeService.save_account(
    token="YOUR_API_TOKEN",
    instance="YOUR_CRN",
    overwrite=True
)

Modalità simulatore

Attivi per eseguire i notebook senza un account IBM Quantum. Tutte le chiamate QiskitRuntimeService vengono reindirizzate a un simulatore locale. Nessuna modifica delle celle necessaria. Questo si applica solo all'esecuzione di codice incorporata su questo sito — l'apertura di un notebook in JupyterLab utilizza il runtime Qiskit standard.

I circuiti transpilati e i risultati specifici del backend differiranno dall'hardware reale quando si utilizza la modalità simulatore. Gli output statici attesi mostrati sulle pagine riflettono i backend IBM reali.

Progressi di apprendimento

I suoi progressi di lettura ed esecuzione sono tracciati localmente nel suo browser. Le pagine visitate mostrano un ✓ nella barra laterale; i notebook eseguiti mostrano un ▶.

Nessun progresso tracciato ancora. Visiti tutorial e guide per iniziare il tracciamento.

Output pre-calcolati

Ogni pagina di notebook mostra output pre-calcolati (immagini, tabelle, testo) dalle esecuzioni originali di IBM. Quando clicca Run per eseguire codice live, sia gli output originali che i suoi nuovi risultati live vengono mostrati affiancati. Attivi questo toggle per nascondere gli output originali durante l'esecuzione live, mantenendo visibili solo i suoi risultati.

Avvisi Python

Per impostazione predefinita, gli avvisi Python (avvisi di deprecazione, suggerimenti runtime) sono soppressi per un output del notebook più pulito. Disattivi questo per vedere tutti gli avvisi — utile per il debugging o per apprendere le modifiche API.

Pacchetti Binder

Quando è in esecuzione su GitHub Pages, il codice viene eseguito tramite MyBinder. L'ambiente Binder include i pacchetti Qiskit principali pre-installati:

Aiuto configurazione

Configurazione RasQberry

Se sta eseguendo su un RasQberry Pi, il server Jupyter dovrebbe essere rilevato automaticamente. In caso contrario, si assicuri che il servizio jupyter-tutorials sia in esecuzione:

sudo systemctl status jupyter-tutorials

Configurazione Jupyter locale

Avvii un server Jupyter con CORS abilitato:

jupyter server --ServerApp.token='rasqberry' \
  --ServerApp.allow_origin='*' \
  --ServerApp.disable_check_xsrf=True

Configurazione Docker

Il container Docker genera un token Jupyter casuale all'avvio. L'esecuzione di codice tramite il sito web (porta 8080) funziona automaticamente — nessun token necessario. Il token è richiesto solo per l'accesso diretto a JupyterLab sulla porta 8888.

Per recuperare il token dai log del container:

docker compose --profile jupyter logs | grep "Jupyter token"

Per impostare un token fisso:

JUPYTER_TOKEN=mytoken docker compose --profile jupyter up

Server remoto

Per server remoti, si assicuri che CORS sia configurato per consentire connessioni da questo sito. Aggiunga quanto segue al suo jupyter_server_config.py:

c.ServerApp.allow_origin = '*'
c.ServerApp.allow_credentials = True
